Бот фг

1. Инит чата. Админ создает чат, первым делом добавляет туда бота. Админ пишет команду /init_fg. После этого в базу
   сохраняется информация по данному чату ФГ. 
   - Сохраняем название чата
   - ID чата 
   - ID админа который этот чат создал
2. При отправке сообщения с хэштегом #отчет необходимо сохранять отчет пользователя. Бот должен ответить "Отчет принят"
   - Бот смотрит ID чата отправки, находит по нему фг
   - Сохраняет данные пользователя если раньше этого не сделал, то есть делает useart
   - Добавляет в таблицу reports запись о созданном отчете, с ссылкой на сообщение
3. При команде /fg_statistic пользователь отправивший команду получает отчет по всем пользователям ФГ чата отправили они
   отчет или нет за текущий день. При этом нужно считать только по идущей ФГ, то есть по той, которая создана позже
   всего
   - Бот берет ID пользователя
   - Находит его как admin_id в таблице fg и берет самый новый fg
   - Получает статистику по конкретному fg_id за сегодня
4. При команде /fg_stat_all пользователь отправивший команду получает статистику по всем участникам за весь период ФГ.
   Админ получает по каждому участнику количество отчетов в этом ФГ, дальше расшифровка и детальное описание в какую
   дату отчет был отправлен с датами по от более свежик к более старым
   - Бот берет ID пользователя
   - Находит его как admin_id в таблице fg и берет самый новый fg
   - Получает статистику по конкретному fg_id за все дни и формирует больше сообщение - отчет. 
   - Отчет это просто сообщение текстовое.
   - В отчете участники расположены в порядке убывания количества отчетов

Текст отчета: 

Отчет по фг 

1. ФИО - {количество} отчетов
   - дата отчета ссылка на сообщение
   - дата ссылка на сообщение
2. ФИО - {количество} отчетов
   - дата ссылка на сообщение
